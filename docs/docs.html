§ 01

Installation

Global install

bash
$ npm install -g @srivtx/tomato-css
$ tomato app.tom -o styles.css

Project install

bash
$ npm install --save-dev @srivtx/tomato-css
$ npx tomato src/app.tom -o dist/styles.css

Or add to package.json:

package.json
{
  "scripts": {
    "build": "tomato src/app.tom -o dist/styles.css",
    "watch": "tomato --watch src/app.tom -o dist/styles.css"
  }
}

No install

bash
$ npx @srivtx/tomato-css app.tom
§ 02

Your first .tom file

Create a file called styles.tom:

styles.tom
button:
  bg blue-500
  color white
  pad 2 4
  round lg
  pointer
  smooth

Compile it:

bash
$ tomato styles.tom -o styles.css

Output is plain CSS you can ship:

styles.css
button {
  background: #3b82f6;
  color: #ffffff;
  padding: 0.5rem 1rem;
  border-radius: 0.5rem;
  cursor: pointer;
  transition: all 0.2s ease;
}
§ 03

CLI usage

bash
$ tomato <input.tom> [-o output.css]
$ tomato --watch <input.tom> [-o output.css]
Command What it does
tomato app.tom Compile to app.css
tomato app.tom -o styles.css Custom output file
tomato --watch app.tom Recompile on file change
tomato --version Print the version

The CLI exits with code 1 on a compile error, so it works as a build step in CI.

§ 04

Selectors

Any selector you'd write in CSS works in Tomato. The compiler is permissive.

HTML elements

styles.tom
button:
  bg blue-500

nav:
  row spread

input:
  pad 2 4

Classes

styles.tom
.btn-primary:
  bg blue-500
  color white

.card:
  bg white
  round xl
  shadow lg

IDs

styles.tom
#header:
  bg slate-900
  pad 4 6

#main-content:
  pad 8

Auto-class conversion

Names that aren't HTML elements become classes automatically:

styles.tom
# these are equivalent
card:          # → .card { }
.card:         # → .card { }

# but these stay as elements
button:       # → button { }
nav:          # → nav { }
Tip. HTML elements like button, nav, div, header, footer stay as element selectors. Everything else becomes a class. Use the dot prefix when you want to be explicit.
§ 05

Components

Define reusable style groups with component name: and pull them in with use name.

Basic components

styles.tom
component btn:
  pad 2 4
  round lg
  bold
  pointer
  smooth
  no border

button:
  use btn
  bg blue-500
  color white

.btn-secondary:
  use btn
  bg violet-500

Nested components

Components can use other components, including their nested pseudo states.

styles.tom
component base-input:
  pad 3 4
  round md
  border 1px solid slate-300
  smooth

component focus-ring:
  use base-input

  focus:
    ring 2px solid blue-400
    border-color blue-500

input:
  use focus-ring
  w-full
Tip. When you use a component, all its properties are expanded inline. You can override them by adding your own after the use statement.

Form input components

styles.tom
component input-base:
  w-full
  pad 3 4
  round lg
  border 1px solid slate-200
  bg white
  smooth

  focus:
    border-color blue-500
    no outline

textarea:
  use input-base
  min-height 120px
  resize vertical

Layout components

styles.tom
component container:
  max-width 1200px
  margin 0 auto
  pad 4 6

component stack:
  column
  gap 4

.page:
  use container
  use stack
  gap 8
Heads up. You can use multiple components with multiple use statements. Properties apply in order, so later ones override earlier ones.
§ 06

Pseudo states

Indent one level. The supported pseudos are hover, focus, active, and disabled.

styles.tom
button:
  bg blue-500
  color white
  smooth

  hover:
    bg blue-600
    shadow lg

  focus:
    ring 2px solid blue-400
    no outline

  active:
    bg blue-700

  disabled:
    opacity 0.5
    cursor not-allowed
§ 07

Responsive design

Match the Tailwind widths by default. Indent one level inside the breakpoint block.

styles.tom
hero:
  pad 16
  grid 3
  gap 6

  @mobile:
    pad 4
    grid 1

  @tablet:
    grid 2
Query Breakpoint
@mobile: max-width: 640px
@tablet: max-width: 768px
@laptop: max-width: 1024px
@desktop: max-width: 1280px
@wide: max-width: 1536px
@dark: prefers-color-scheme: dark
@light: prefers-color-scheme: light
§ 08

Custom tokens

Define a name once, use it everywhere. Tokens are local to the file. Share them across files with @import.

styles.tom
colors:
  primary blue-500
  secondary violet-500
  dark slate-900
  light slate-50
  muted slate-400
  danger red-500
  success green-500

button:
  bg primary
  color light

.alert-danger:
  bg danger
  color white
bash
$ tomato app.tom -o styles.css
# Cannot import "tokens.tom": file not found  # exits 1

Or share tokens across files:

app.tom
@import "./tokens.tom"

.button-primary:
  bg primary
§ 09

Color reference

22 color scales built in (Tailwind's).

styles.tom
# 22 color scales built-in
slate, gray, zinc, neutral, stone
red, orange, amber, yellow, lime
green, emerald, teal, cyan, sky
blue, indigo, violet, purple
fuchsia, pink, rose

# usage
bg rose-500
color slate-900
border 1px solid blue-300

Each scale runs from 50 to 950. You can also pass raw CSS colors when you need something off the palette:

styles.tom
bg #f43f5e
bg rgb(244 63 94 / 0.5)
bg transparent
bg currentColor
